# AlgoTradingTool — A股量化分析工具

> 基于 Flask + akshare + SnowNLP + ECharts 的 A 股市场情绪量化分析平台

输入任意 A 股名称，自动抓取实时交易数据与多平台新闻，通过中文情感分析和量化模型给出**推荐指数**与多维度可视化报告。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 智能搜索 | 输入股票名称或代码，自动匹配 5527 只 A 股 |
| 📊 实时行情 | 股价、涨跌幅、均线系统等交易数据 |
| 📰 多源新闻 | 东方财富财经新闻 + 雪球热帖 + 公告信披 |
| 🧠 情感分析 | SnowNLP 中文自然语言处理，量化每条新闻的情感倾向 |
| 🎯 推荐指数 | 综合财务、情感、风险三维度加权评分（0-100）+ 星级评定 |
| 🕸️ 六维雷达 | 财务表现、资本市场、市场竞争、产品技术、公司经营、行业政策 |
| 🥧 来源饼图 | 官方媒体 / 主流媒体 / 自媒体 三分类置信度分析 |
| 📋 新闻摘要 | 每条新闻一句话摘要，独立卡片展示，带情感标签 |

---

## 技术架构

```
用户输入 → akshare 数据抓取 → SnowNLP 情感分析 → 综合评分引擎 → ECharts 可视化
```

| 环节 | 技术选型 |
|------|---------|
| 后端框架 | Flask |
| 行情数据 | akshare（新浪 + 腾讯双源） |
| 新闻抓取 | 东方财富 API |
| 中文分词 | jieba |
| 情感分析 | SnowNLP |
| 图表渲染 | ECharts 5.x |
| 前端样式 | 深色科技风 CSS |

## 快速开始

### 环境要求

- Python 3.9+
- 网络代理（访问 Sina 股票列表接口需要）

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置代理

编辑 `src/config.py` 中的代理端口：

```python
PROXY_PORT = 6518  # 改为你的代理端口
```

### 3. 启动应用

```bash
python src/app.py
```

### 4. 打开浏览器

访问 **http://127.0.0.1:18000**

## 评分体系

### 推荐指数计算

```
推荐指数 = 财务评分 × 0.4 + 情感评分 × 0.4 + (100 - 风险评分) × 0.2
```

| 指数范围 | 星级 | 建议 |
|----------|------|------|
| ≥ 85 | ★★★★★ | 强烈推荐 |
| 70 – 84 | ★★★★ | 推荐买入 |
| 55 – 69 | ★★★ | 推荐买入 |
| 40 – 54 | ★★ | 谨慎观望 |
| < 40 | ★ | 建议回避 |

### 核心指标

| 指标 | 计算方式 |
|------|---------|
| 📈 财务评分 | PE/PB 估值 + ROE 水平 + 营收增速 + 近期涨跌幅 |
| 💬 情感评分 | SnowNLP 对所有新闻的情感均值（0-100） |
| ⚠️ 风险评分 | 情感分歧度 + 新闻量不足惩罚，越低越好 |
| 🎯 置信度 | 来源可靠性加权（官方 1.0 > 主流 0.75 > 自媒体 0.35） |

### 六维雷达图

| 维度 | 数据来源 |
|------|---------|
| 财务表现 | 实时财务指标（PE / PB / ROE） |
| 资本市场 | 交易数据（涨跌幅 / 量比 / 均线） |
| 市场竞争 | 新闻关键词分类 + 情感聚合 |
| 产品技术 | 新闻关键词分类 + 情感聚合 |
| 公司经营 | 新闻关键词分类 + 情感聚合 |
| 行业政策 | 新闻关键词分类 + 情感聚合 |
